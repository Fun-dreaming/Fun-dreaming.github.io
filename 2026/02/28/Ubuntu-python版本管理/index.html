Ubuntu下的Python版本管理-使用update-alternatives

发表于 2026-02-28

在嵌入式开发中，经常会使用到python作为构建辅助工具。不同的SDK可能会使用不同的python版本，而python2与python3又往往存在兼容性问题。因此，在开发环境中需要进行python版本管理。本文将介绍Ubuntu下使用update-alternatives的版本管理方法。

1.查看系统已安装的Python版本

在Ubuntu中，python往往指代pyhton2,python3指代python3，可以用过python(3) --version指令查看当前版本

也可以使用whereis python来定位所有python可执行文件的路径

2.使用update-alternatives进行版本管理

2.1update-alternatives的工作原理

update-alternatives的核心工作原理是基于软链接的层级管理。

当在终端中输入python时，系统首先在环境变量中查找到/usr/bin/python，而/usr/bin/python并不是一个可执行文件，而是一个指向/etc/alternatives/python的软链接，我们可以使用如下指令验证：

而/etc/alternatives/python同样也不是一个可执行文件，其通过软链接指向了真正的可执行文件，比如/usr/bin/python2.7：

因此，update-alternatives 的工作原理就是自动管理和修改 /etc/alternatives/ 目录下的这层软链接，进而切换python的指向。
